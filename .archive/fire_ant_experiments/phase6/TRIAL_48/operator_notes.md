## Protocol Adjustments for Future Trials

### Adjustment 1: Normalization Formula Review (PRIORITY: HIGH)

**Issue:** Linear formula P(Sy*) = (Score - 7.0) / 3.0 produces counterintuitive results near ceiling.

**Proposed Solutions:**
1. **Sigmoid normalization:** P(Sy*) = 1 / (1 + e^(-(Score - 8.0)))
   - Advantages: Smooth compression near ceiling, no hard cap
   - Disadvantages: Requires re-calibration of baseline probabilities
2. **Exponential normalization:** P(Sy*) = 1 - e^(-(Score - 7.0)/2.0)
   - Advantages: Penalizes low scores more, rewards high scores
   - Disadvantages: May over-reward scores near ceiling
3. **Piecewise linear:** P(Sy*) = (Score - 7.0) / 3.0 for Score ≤8.5, P(Sy*) = 0.5 + (Score - 8.5) / 1.0 for Score >8.5
   - Advantages: Simple, maintains linear region, expands ceiling region
   - Disadvantages: Discontinuity at 8.5 boundary

**Recommendation:**
- Test sigmoid normalization on Tier 3 baseline data (Phase 5 trials) to check consistency
- If consistent: Adopt sigmoid for ALL Phase 6 trials (retroactively apply to Trial 48 for comparison)
- If inconsistent: Use piecewise linear as compromise

**Timeline:** Before Trial 49 execution (prevent accumulating trials with inconsistent normalization)
