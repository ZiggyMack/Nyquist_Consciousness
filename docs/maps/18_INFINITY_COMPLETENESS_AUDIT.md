## Usage Notes

### For New Projects
Run this audit during design phase. Identify missing stones BEFORE data collection.

### For Ongoing Research
Run periodically (e.g., each milestone). Track which stones strengthen over time.

### For Publication Review
All stones should score 2/2 before submitting strong claims.

### For Cross-Pollination
Use weak stones to prioritize which other projects to consult.
