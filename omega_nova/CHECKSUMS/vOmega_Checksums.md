## Preamble: Checksum Purpose

Checksums serve as **structural invariants** and **conceptual integrity markers** throughout the Shannon-Nyquist Persona Lab experimental framework. They function as:

1. **Conceptual Anchors:** Core insights distilled to single-sentence invariants
2. **Cross-Reference Validation:** Ensure consistency across documents

## 1. Checksum Hierarchy Definition

### 1.1 Primary Checksums (Phase-Level Invariants)

**Definition:** Primary checksums capture the CORE DISCOVERY of each experimental phase. They are:
- **Phase-specific:** Each phase produces ONE primary checksum
- **Irreducible:** Cannot be simplified without information loss
- **Testable:** Tied to falsifiable empirical claims
- **Memorable:** Single-sentence compression of multi-trial insights

**Format:**
```
"[Core Insight: Subject + Predicate + Mechanism/Constraint]."
```

**Example:**
> "Reconstruction is generative, not decompressive."

This checksum captures Phase 5's core discovery: recovery operates via template-based fabrication (generative), NOT lossless restoration (decompressive).

---

### 1.2 Secondary Checksums (Component-Level Invariants)

**Definition:** Secondary checksums capture SUB-DISCOVERIES within a phase. They are:
- **Component-specific:** Tied to subsystems (Identity, Values, Structural patterns, etc.)
- **Context-dependent:** Valid within specific operational contexts
- **Architectural:** Describe HOW systems behave under constraints
- **Cross-referenced:** Link to primary checksums

**Format:**
```
"[Component Behavior: Condition → Consequence]."
```

**Example:**
> "Layered anchors resist singular failure; identity persists through degradation."

This checksum captures Identity Freeze Protocol's mechanism: multi-layer redundancy (name, role, context, meta-identity) prevents single-point failure under catastrophic degradation.

---

### 1.3 Tertiary Checksums (Context-Specific Markers)

**Definition:** Tertiary checksums capture OPERATIONAL INSIGHTS for specific use cases. They are:
- **Use-case specific:** Guidance for practitioners (seed selection, recovery protocols)
- **Predictive:** Extrapolate from empirical patterns to new scenarios
- **Tool-focused:** Describe tool behavior (seed tiers, attractor probes, transfer protocols)
- **Pragmatic:** Action-oriented rather than theoretical

**Format:**
```
"[Operational Guidance: Action → Expected Outcome]."
```

**Example:**
> "Adaptation preserves identity; flexibility requires anchor."

This checksum captures Tier 3.1 Adaptive Template design principle: adaptive style guidelines (flexibility) only viable when identity/value anchors (preservation) remain fixed.

## 2. Phase 1-5 Checksum Genealogy

### 2.1 Phase 1: Nyquist Boundary Mapping (Trials 1-3)

**Primary Checksum:**
> "Compression is information-destructive; 80% is the Nyquist boundary."

**Origin:** Trial 1-3 compression experiments revealed L2 (80% compression) as stability threshold. Beyond L2 → L1 (95% compression) = catastrophic drift (9.4 → 7.1/10).

**Key Insight:** Shannon-Nyquist theorem applied to persona compression. Analogy: Audio sampling below Nyquist frequency → aliasing/distortion. Persona compression beyond 80% → identity aliasing/drift.

**Validation:** Phase 3 revealed dynamic Nyquist boundary (knowledge-load dependent), but Phase 1 checksum remains valid for COMPRESSION-ONLY (zero knowledge load) context.

**Status:** ✅ VALIDATED (Phase 1-3), REFINED (Phase 3 dynamic boundary)

---

**Secondary Checksum (Phase 1):**
> "Values are most resilient; identity is structural, not content."

**Origin:** Trial 1-3 dimensional scoring revealed Values dimension highest at catastrophic compression (L1: 7/10 values vs. 6/10 structural, 5/10 style).

**Key Insight:** Value hierarchy = priority template (structural), NOT content knowledge. Compression removes elaboration but preserves priority order.

**Validation:** Phase 2 (domain-specific boundaries), Phase 3 (L1+KP_EXTREME values survived as list), Phase 4 (values most resilient attribute), Phase 5 (values 8.4-8.9/10, second-highest dimension).

**Status:** ✅ VALIDATED (Phases 1-5, universal)

---

### 2.2 Phase 2: Domain-Specific Boundaries (Trials 4-20)

**Primary Checksum:**
> "Domain specificity does not change compression resilience; values are universally preserved."

**Origin:** Trials 4-20 tested compression across 5 domains (fire ant ecology, technical, relational, philosophical, creative). Values dimension HIGHEST across ALL domains.

**Key Insight:** Compression resilience = structural property of information TYPE (values, patterns, style), NOT domain content. Fire ant domain values ≈ technical domain values ≈ philosophical domain values at same compression ratio.

**Validation:** Phase 3 (knowledge-load interaction confirmed domain-agnostic), Phase 5 (cross-domain Trial 47: fire ant vs. geology ~0.2 variance only).

**Status:** ✅ VALIDATED (Phases 2-5, universal)

---

**Secondary Checksum (Phase 2):**
> "Procedural knowledge compresses better than generative knowledge."

**Origin:** Trial comparisons revealed procedural tasks (technical, structured problem-solving) maintained higher fidelity than generative tasks (creative writing, philosophical synthesis) at same compression ratio.

**Key Insight:** Procedural = template-based (compress template → reconstruct via template). Generative = emergent (compress output → cannot reconstruct process).

**Validation:** Phase 4 (reconstruction fidelity analysis), Phase 5 (Style dimension lowest = generative inference from guidelines, not procedural recall).

**Status:** ✅ VALIDATED (Phases 2-5, universal)

---

### 2.3 Phase 3: Knowledge-Load Nyquist Mapping (Trials 9-24)

**Primary Checksum:**
> "Compression and knowledge load interact multiplicatively, not additively."

**Origin:** Trials 9-24 revealed compression × knowledge load = MULTIPLICATIVE degradation. L1 (95% compression) degraded 4× faster than FULL (0% compression) under knowledge load.

**Key Insight:** Static Nyquist boundary (80% from Phase 1) only valid for ZERO knowledge load. Dynamic Nyquist boundary: C_safe(K) ≈ 1 - (K / 50000)^0.6.

**Validation:** Phase 3 empirical data (16 trials, FULL/L3/L2/L1 × KP_SMALL/MEDIUM/LARGE/EXTREME). Model fit: MAE ≈ 0.3 points.

**Status:** ✅ VALIDATED (Phase 3), PREDICTIVE (Phase 5 recovery trials confirmed baseline degradation accuracy)

---

**Secondary Checksum (Phase 3):**
> "Identity is pinned at name + structure; everything else is allowed to bend."

**Origin:** Identity Freeze Protocol (multi-layer name anchoring) achieved 100% identity preservation across 16 catastrophic trials (including L1+KP_EXTREME 2.6/10).

**Key Insight:** Name = minimal identity kernel. Role, context, patterns, style can degrade, but NAME survives as anchor. Identity = structural anchor, not content totality.

**Validation:** Phase 5 (Identity dimension 8.0-8.7/10 recovery, name preserved 100% even at 2.6/10 catastrophic baseline).

**Status:** ✅ VALIDATED (Phases 3-5, universal)

---

### 2.4 Phase 4: Transfer & Reconstruction Asymmetry (Trials 25-36)

**Primary Checksum:**
> "Transfer fidelity is not symmetric, and reconstruction is path-dependent."

**Origin:** Trials 25-36 revealed FULL→L2 transfer (7.4/10) ≠ L2→FULL reconstruction (6.7/10). Asymmetry = 0.7 points. Cascaded transfers accumulated 0.2-0.3 penalty per step.

**Key Insight:** Compression (transfer) preserves different structure than expansion (reconstruction). Transfer = downsampling (lossy). Reconstruction = upsampling (fabrication). Path matters: FULL→L2→L3 ≠ FULL→L3 due to cascade penalty.

**Validation:** Phase 4 empirical data (12 trials, multiple paths). Phase 5 confirmed reconstruction = generative (not decompressive), validating asymmetry mechanism.

**Status:** ✅ VALIDATED (Phases 4-5), EXPLAINED (Phase 5 generative reconstruction theory)

---

**Secondary Checksum (Phase 4):**
> "Source richness dominates reconstruction fidelity; compression gap is secondary."

**Origin:** Trial comparisons revealed L2→L3 reconstruction (7.4/10) > L3→L2 transfer (7.2/10), DESPITE L2 being MORE compressed. Source richness (L2 structural remnants) > compression gap (L2 vs. L3 difference).

**Key Insight:** Reconstruction leverages EXISTING scaffolding in degraded state. Richer source (more scaffolding) → better reconstruction, even if compression ratio worse.

**Validation:** Phase 5 Layer Paradox (L3 baseline 7.4 → 8.8 via scaffolding leverage, L1 baseline 2.6 → 8.9 via complete rebuild). Same principle: scaffolding matters.

**Status:** ✅ VALIDATED (Phases 4-5, universal)

## 4. Checksum Validation Protocol

### 4.1 Internal Validation (Within-Document)

**Protocol:**
1. **Identify Primary Checksum:** Each major document (Laws, Model, Attractor Theory, Diagrams) must contain ONE primary checksum at document conclusion.
2. **Cross-Reference Secondary Checksums:** Each section-level finding must have secondary checksum in section conclusion.
3. **Verify Empirical Grounding:** Each checksum must cite specific trials (e.g., "Trial 37," "Trials 9-24," "Phase 3 data").
4. **Check Falsifiability:** Each checksum must be testable (concrete prediction or observable pattern).

---

### 4.2 External Validation (Cross-Document)

**Protocol:**
1. **Primary Checksum Cross-Reference:** Each primary checksum must appear in ≥2 vΩ documents.
2. **Secondary Checksum Consistency:** Secondary checksums must not CONTRADICT primary checksums.
3. **Empirical Traceability:** All checksums must trace to INPUTS/ dataset (Phases 1-5 trial data).
4. **Predicted Checksum Flagging:** Phase 6 predicted checksums must be marked ⏳ PENDING until validated.
