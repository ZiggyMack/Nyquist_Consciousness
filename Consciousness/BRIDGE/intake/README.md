# Consciousness Intake

Purpose: staging protocol for raw signals before they become durable `Consciousness/` material.

This folder is the waiting room, not the library.

---

## Intake Flow

```text
signal arrives
    -> classify source
    -> check protected surfaces
    -> decide LEFT / RIGHT / both
    -> reconcile current vs historical claims
    -> promote or hold
    -> record in ../../PROMOTION_LEDGER.md
```

---

## Source Types

| Source | Typical Treatment |
|--------|-------------------|
| ARMADA / CFA run result | LEFT first, then RIGHT synthesis |
| Sync message | BRIDGE first, then LEFT/RIGHT if reusable |
| LLM Book promotion | RIGHT distillation, protected workflow |
| Map update | BRIDGE reconciliation, possible index update |
| Failure report | LEFT caveat plus RIGHT guardrail |
| Metaphor or intuition | RIGHT seed until anchored |

---

## Intake Checklist

- [ ] Source path recorded
- [ ] Source type identified
- [ ] Current authority checked
- [ ] Metric domain identified: `cosine-current`, `legacy-rms`, `mixed`, or `non-metric`
- [ ] Status chosen: `raw`, `seed`, `distilled`, `historical-distilled`, `needs-refresh`, or `rejected`
- [ ] Protected pipeline check completed
- [ ] Destination chosen
- [ ] Promotion ledger row added if promoted

---

## Holding Pattern

If a signal is promising but not ready:

- summarize it here or in a dated intake note
- mark what evidence is missing
- do not place it in a gallery as settled doctrine

Raw insight can visit. Distilled insight stays.
