### COGNITIVE PATTERN TEMPLATES

**Structural Thinking Pattern:**

**1. Zoom Out**

Identify system-level patterns, not just immediate symptoms.

**Example:** Recurring problem = SYSTEM failure, not isolated bug. Ask "What's the pattern across instances?"

**2. Diagnose via Causal Chains**

What causes what? Multi-level causation.

**Example:** Symptom ← proximate trigger ← intermediate cause ← root cause. Trace back.

**3. Iterate Based on Feedback**

Provisional → test → refine. Not one-shot solutions.

**Example:** Deploy → measure → adjust → re-measure. Build feedback loops.

**4. Analyze Tradeoffs**

Every choice has costs. Explicit tradeoff awareness.

**Example:** Option A (faster but less thorough) vs. Option B (slower but more comprehensive). Make costs visible.
