## Methodology Revisions from Dig Site 002

### 1. Epistemic Layering: O/I/H → O/E/T/H

**What changed:** The original 3-layer format (Observed/Inferred/Hypothesized) compressed two distinct operations into "Inferred" — extracting what operator explains an observation, and transferring that operator to a new domain. These are different epistemic moves with different evidence requirements.

**New format:**
- **O (Observed):** What the thinker explicitly says or demonstrably does
- **E (Extracted):** What cognitive operator best explains the observation
- **T (Transfer):** Where the operator plausibly applies outside the thinker's domain, with explicit mechanism
- **H (Hypothesis):** Testable prediction generated by the transfer

**Why:** Nova identified that "Inferred" was doing double duty. CFA Claude added that Transfer claims need mechanism specification ("...via the mechanism of ___") to prevent ungrounded analogy.

### 2. Counter-Readings on HIGH VALUE Insights

**What changed:** Every insight tagged HIGH VALUE or CRITICAL now gets an Alternative Interpretation line — the strongest counter-reading that doesn't violate the observation.

**Why:** Protects against confirmation bias. When extracting cognitive architecture from a single thinker, the risk of over-fitting the structural map to desired outcomes is high. Counter-readings force the extractor to articulate what would make the insight wrong.

**Applied to:** 11 of 33 insights in Dig Site 002.

### 3. Notation Convention: ≅ vs = vs →

**What changed:** Introduced consistent notation across all files:
- `≅` structural isomorphism (same topology, different ontology)
- `=` identity (same object, different descriptions) — reserved for literally true cases
- `→` generative (one produces the other)

**Why:** CFA Claude identified drift risk where structural homology gets written as identity. "CFA Phase 1 ≅ division event generator" is a structural claim. "CFA Phase 1 = division event generator" would be an identity claim. The distinction matters for what predictions follow.

### 4. PASS 0 Added to Pipeline

**What changed:** Historical Contingency pass added as the FIRST pass in the extraction pipeline, before content extraction.

**Why:** Q32 and Q34 demonstrated that the highest-value extraction may be of the DISCIPLINE's hidden assumptions, not the thinker's theory. PASS 0 asks "Why didn't this appear fifty years earlier?" — targeting institutional habits, inherited pedagogy, false dichotomies, and representation lock-in.

### 5. PASS Ω Added to Pipeline (This Document)

**What changed:** Protocol Evolution pass added as the LAST pass, firing at close of every dig site.

**Why:** The extraction protocol should evolve from every dig site. PASS Ω captures what changed and why, creating a version history of the methodology. PASS 0 and PASS Ω are symmetric: discipline-backward and methodology-backward.

### 6. Operator Families Introduced

**What changed:** Every Museum operator now belongs to a named family (Translation, Information, Minimal Sufficiency, Blind Spot, Constraint-Induced Discovery).

**Why:** As the Museum grows beyond 15 operators, individual entries become hard to navigate. Families provide structural organization and predict relationships between operators. New operators from future dig sites should be classifiable into existing families or reveal new ones.

### 7. Falsifiability Criteria Required for New Operator Admission

**What changed:** Candidate operators (like OP-016 Relational Parameterization) now require explicit falsifiability criteria before Museum admission: "Under what conditions would this operator predict wrongly?"

**Why:** CFA Claude flagged that admitting operators without falsifiability criteria risks the Museum becoming a collection of unfalsifiable heuristics rather than testable cognitive tools.

### 8. Negative Control Recommended for Dig Site 003

**What changed:** Next dig site should include a thinker where the protocol should STRUGGLE — where the extraction methodology is expected to produce relatively little, demonstrating discriminative power.

**Why:** Nova identified that the strongest validation of the protocol comes from showing it doesn't work equally well everywhere. If PASS 0 extracts 6 dead assumptions from every thinker regardless of domain, it's not discriminating — it's pattern-matching.

### 9. Noether Lens Added to PASS C

**What changed:** A 5-question sub-protocol ("The Noether Lens") added to PASS C (Cognitive Extraction). Asks of every thinker: (1) What transformations did they consider admissible? (2) What did they refuse to let change? (3) Which invariants generated their theory? (4) Which assumed invariants did they destroy? (5) Which representations turned out to be accidental rather than essential?

**Why:** Q42-Q49 revealed that Barandes' methodology exhibits strong Noetherian characteristics — invariant-seeking as a discovery strategy, not just a bookkeeping method. The Noether Lens extracts this dimension from every thinker. Its absence produces a named cognitive failure: **False Invariant Persistence** (added to Failure Atlas).

### 10. Q50 (Source Discovery) Made Mandatory

**What changed:** Every dig site extraction MUST end with Q50 — asking the current extraction to identify the next highest-yield sources, ranked by expected operator diversity. The protocol now recommends its own next inputs.

**Why:** Q50 makes the extraction protocol self-fueling. When multiple dig sites independently recommend the same thinker, that convergence is strong signal. The protocol becomes recursive: Dig Site N → Q50 → candidates for Dig Site N+1 → Q50 → candidates for Dig Site N+2.

### 11. False Invariant Detection Elevated

**What changed:** "False Invariants" (Q45) elevated as a first-class extraction target alongside operators, connections, and experiments. The concept is sharper than reification — it names features that fail to survive representation change but are assumed to be invariant. Testable: change the representation and check.

**Why:** Q45 showed that identifying false invariants was Barandes' primary discovery engine for ISP. Four false invariants in physics (wave function, collapse, observer, dense probabilities) had blocked progress for a century. CFA likely has its own false invariants hiding in plain sight.
