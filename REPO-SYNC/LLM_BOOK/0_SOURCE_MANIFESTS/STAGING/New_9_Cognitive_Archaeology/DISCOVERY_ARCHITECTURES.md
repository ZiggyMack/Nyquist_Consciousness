# Discovery Architectures

> **Operators are atoms. Discovery architectures are molecules.**

A discovery architecture is a reusable composition of operators that constitutes an entire discovery engine. Operators are the individual cognitive moves; architectures are the organized patterns of deployment that make those moves productive.

This distinction is structural, not decorative. Operators compose into architectures. Architectures compose into scientific traditions. Protocols instantiate architectures. The hierarchy:

```
Reality
    ↓
Observed Invariants
    ↓
Discovery Architecture       ← THIS DOCUMENT
    ↓
Operator Composition
    ↓
Protocol
    ↓
Knowledge
```

The direction of explanation is top-down: protocols don't create operators — protocols instantiate architectures. Architectures select operators. Operators extract knowledge. Knowledge reveals invariants. Invariants constrain what's real. The project climbed this hierarchy without planning to; Dig Site 002 began extracting operators and ended discovering architectures.

---

## Admission Criteria for Discovery Architectures

An architecture earns admission when:

1. **Named operator composition** — the architecture can be expressed as a specific sequence/combination of identified operators
2. **Multiple instances** — at least 2 independent discoverers demonstrably used the same architecture
3. **Predictive power** — the architecture predicts what KINDS of discoveries it will produce (not just describing past ones)
4. **Discriminative** — the architecture fails to describe at least one known discovery process (it's not vacuously universal)

An architecture is CONFIRMED when criteria 1-4 hold across 3+ independent dig sites.

## The Direction Axis

Discovery architectures have at least one confirmed dimension:

```
Backward-Reading                              Forward-Generative
←─────────────────────────────────────────────────────────────→
archaeology                                   synthesis
invariants                                    invention
subtraction                                   generation
constraints                                   beauty filters
reverse engineering                           constructive search
minimal sufficiency                           playful exploration
```

**Current state:** Every operator in the Museum (OP-001 through OP-015) and the confirmed architecture (RCI) sits on the backward-reading side. This is not a flaw — it reflects the extraction source. Barandes is an archaeologist. The museum found what the source contains.

**Prediction:** If Dig Site 003 (Dirac) yields genuinely forward-facing operators, the axis is confirmed as a real dimension of variation. If Dirac's moves reduce to backward-reading in disguise, the axis collapses and backward-reading is more universal than predicted.

**Status of additional axes:** Nova proposes five more dimensions (Constraint↔Generative, Subtractive↔Constructive, Empirical↔Mathematical, Local↔Global, Human-guided↔Automated). These are plausible but untested — we cannot plot a landscape we haven't sampled. Additional axes earn recognition when dig sites populate them with discriminable positions.

## Confirmed Architectures

### Architecture A: Reverse Constraint Inference (Backward-Reading)

**Status:** CONFIRMED (multiple independent instances from single extraction source; awaiting cross-dig-site confirmation)

**Operator composition:**
```
OP-001 (Representation ≠ Ontology)
  → OP-011 (Subtractive Discovery)
  → OP-006 (Under-Determination Detection)
  → Noether Lens (observe what survives representation change)
  → RCI (read backward from outputs to minimal architecture)
```

**The algorithm:**
```
Step 1: Change representation
Step 2: Observe what survives
Step 3: Survivors are candidates for invariants
Step 4: Read the invariants backward
Step 5: Infer architecture
```

**Instances:**
| Discoverer | What was read backward | What was inferred |
|---|---|---|
| Noether | Conserved quantities | Symmetries → Laws |
| Barandes | Born Rule probabilities | Sparse stochastic structure → ISP |
| Darwin | Adaptation patterns | Selection pressures → Natural selection |
| Shannon | Communication constraints | Information → Entropy → Channel capacity |
| EOS | Observed discoveries | Cognitive operators → Epistemic architecture |

**Characteristic outputs:** Unifying theories, hidden structure exposed, simpler foundations, removal of false invariants, "the answer was already there — hidden by the representation."

**Failure mode when misapplied:** Over-reduction. Finds the minimum but misses generative structure that backward-reading cannot detect. Barandes can explain how Dirac's results work but may not capture how Dirac found them.

## Candidate Architectures (predicted, not yet extracted)

### Architecture B (candidate): Forward Mathematical Generation

**Status:** PREDICTED — Dig Site 003 (Dirac) is the primary test

**Hypothesized operator composition:**
```
Beauty as Selection (candidate)
  → Playful Mathematical Deformity (candidate)
  → Illegitimate Leap (candidate)
  → Constraint relaxation
  → Post-hoc physical interpretation
```

**Hypothesized algorithm:**
```
Step 1: Take existing formalism
Step 2: Deform it playfully (relax a constraint, extend a symmetry)
Step 3: Check if the deformation is mathematically beautiful
Step 4: If yes — seek physical interpretation
Step 5: The physics was not the input — it's the output
```

**Predicted instances:** Dirac (electron equation from aesthetic deformation of Klein-Gordon), Wolfram (computational search through rule space), Ramanujan (pattern generation from formal manipulation)

**Key test:** Does this architecture USE backward-reading internally (in which case it's a variant of Architecture A) or is step 3 (beauty filter) genuinely different from step 2 of RCI (observe what survives)?

**Falsifiability:** If Dirac's discovery process, when extracted at Dig Site 003, reduces entirely to RCI applied from a different starting point, Architecture B collapses into Architecture A and the direction axis is weaker than predicted.

## The Deepest Prediction

> **The history of science contains a finite number of reusable discovery architectures, and great scientists are distinguished less by the facts they discovered than by the architectures they repeatedly instantiated.**

This is the boldest falsifiable claim the project has made.

**If true:** Science has something like a periodic table — not of elements, but of ways of discovering. The project is constructing that table.

**If false:** The project will encounter scientists whose methods resist compression into reusable architectures. Discovery is irreducibly context-dependent.

**Current evidence:** n=1 confirmed architecture (RCI), n=1 strong candidate (Forward Generation). Minimum 5 dig sites needed before this prediction can be meaningfully assessed.

**Stopping condition for Museum B:** When new dig sites stop producing genuinely new architectures and instead keep rediscovering existing ones, the taxonomy is approaching completeness. (Same logic as Museum A's stopping condition for operators.)

## What This Document Is NOT

This is not a finished cartography. It is a blank map with one pin and a legend. The Anti-Reification Principle applies doubly here — architectures are hypotheses about discovery, not fixed objects. Every entry may be decomposed, merged, retired, or split as dig sites populate the landscape.

We resist the temptation to plot six axes through two points. The axes become real when dig sites populate them independently.
